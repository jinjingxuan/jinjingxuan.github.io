手写vue响应式

Vue.js

  • 负责接收初始化的参数
  • 负责把data中的属性注入到Vue实例，转换成getter/setter
  • 负责调用observer监听data中所有属性的变化
  • 负责调用complier解析指令/差值表达式

Observer.js

  • 负责把data选项中的属性转换成响应式数据
  • data中的某个属性也是对象，把该属性转换成响应式数据
  • 数据变化发送通知
  • observe这个函数传入一个 data（需要被追踪变化的对象），通过遍历所有属性的方式对该对象的每一个属性都通过 defineReactive 处理,以此来达到实现侦测对象变化。
  • 值得注意的是，walk 会进行递归调用。
  • 主要作用是做数据劫持，在数据获得更新的时候（拦截set方法），执行（Dep）的notify方法，通知所有的订阅者（Watcher）。

complier.js

  • 负责编译模板，解析指令/差值表达式
  • 负责页面的首次渲染
  • 当数据变化后重新渲染视图

Dep.js

  • 收集依赖，添加观察者（watcher）
  • 通知所有观察者

watcher.js

  • 在compiler中操作DOM渲染数据时创建Watcher
  • 自身实例化的时候往dep对象中添加自己
  • 当数据变化触发依赖，dep通知所有的watcher更新视图
  • 在执行构造函数的时候将 Dep.target 指向自身，从而使得收集到了对应的 Watcher，在派发更新的时候取出对应的 Watcher ,然后执行 update 函数。
  • cb参数是一个回调函数，指明不同的watcher有不同的更新视图方法

总结

  • 在 new Vue() 后， Vue 会调用_init 函数进行初始化，也就是init 过程，在 这个过程Data通过Observer转换成了getter/setter的形式，来对数据追踪变化，当被设置的对象被读取的时候会执行getter 函数，而在当被赋值的时候会执行 setter函数。
  • 当外界通过Watcher读取数据时，会触发getter从而将Watcher添加到依赖中。
  • 在修改对象的值的时候，会触发对应的setter， setter通知之前依赖收集得到的 Dep 中的每一个 Watcher，告诉它们自己的值改变了，需要重新渲染视图。这时候这些 Watcher就会开始调用 update 来更新视图。
  • 每个组件实例都对应一个 watcher 实例，Vue源码中的Watcher是在组件的mount过程中初始化的
  • 个人理解：比如有一个Vue组件读取了data中的value值，这个时候在getter中Dep就收集了Watcher依赖，当value值改变的时候，触发依赖，执行Watcher中的update方法更新视图。

对于对象

Vue 无法检测 property 的添加或移除。这是因为 Vue 通过Object.defineProperty来将对象的key转换成getter/setter的形式来追踪变化，但getter/setter只能追踪一个数据是否被修改，无法追踪新增属性和删除属性。由于 Vue 会在初始化实例时对 property 执行 getter/setter 转化，所以 property 必须在 data 对象上存在才能让 Vue 将它转换为响应式的。例如：

对于数组

Object.defineProperty 不能监听数组的一些方法，push/pop/splice等等改变原数组的方法不能触发set，需要进行数组方法的重写。（Vue3中使用proxy则不存在这些问题）

此外 Vue 不能检测以下数组的变动：

  1. 当你利用索引直接设置一个数组项时，例如：vm.items[indexOfItem] = newValue
  2. 当你修改数组的长度时，例如：vm.items.length = newLength
